# 基于 STM32F103 + BQ76920 的锂电池 BMS 固件开发

本项目是一个基于 STM32F103C8T6、FreeRTOS 和 TI BQ76920 模拟前端的锂电池 BMS 固件，用于电池数据采样、状态监测、保护判断、充放电控制以及 CAN/UART 调试通信。当前工程按 5 串电芯、1 路温度采样编译，驱动接口沿用 BQ769X0 命名。

## 硬件与软件平台

- 主控芯片：STM32F103C8T6，Cortex-M3，72 MHz
- 电池模拟前端：TI BQ76920；当前配置为 5 串电芯、1 路温度采样
- 实时操作系统：FreeRTOS，基于 CMSIS-RTOS v2
- 开发工具链：Keil MDK-ARM / ARM Compiler 5
- 调试串口：USART1，115200 baud
- CAN 通信：CAN1，125 kbit/s

## 主要功能

- 通过 PB13/PB14 软件 I2C 访问 BQ76920，支持 ACK 检查、CRC8、启动阶段 9-clock 总线恢复和多任务互斥访问
- 初始化时读取 ADC Gain/Offset，校验保护阈值的可编码范围；阈值写入后回读寄存器并反算实际物理值
- 默认软件欠压阈值为 3.10 V，硬件欠压后备请求值为 2.80 V，硬件过压请求值为 4.20 V；芯片校准与量化后的实际值以启动回读日志为准
- 采集电芯电压、电流、温度、包电压、最大值、最小值、压差和平均值等监测数据
- 实现软件过压、欠压、放电过流、短路、充电过流以及充放电高低温保护；连续 3 次监测无效时进入 fail-safe 并关闭 CHG、DSG 和均衡输出
- BQ 硬件 ALERT 故障会在清除 SYS_STAT 的 W1C 位前进行运行期锁存，并立即执行安全关断
- 硬件 OCD/SCD 故障仅在 3 秒冷却、安全关断、连续确认负载移除且 SYS_STAT 无故障后允许显式恢复
- DEVICE_XREADY 在运行期保持锁存；受控重启会先安全关断，再重新加载并验证完整保护配置
- CHG/DSG 默认不自动开启，开启请求必须通过软件保护、硬件故障、采样有效性和包电压检查，写入后还会回读确认；保护激活时强制清除均衡输出
- 提供基于库仑积分、静置 OCV 修正和温度容量折算的 SOC 原型；当前 2.2 Ah 容量参数和 OCV 表仍需结合实际电芯标定
- 提供 UART 命令行调试，支持状态、采样、保护信息查看和受门禁约束的手动控制
- CAN1 每 150 ms 发送 `0x180` 状态帧并接收 `0x300` 命令帧；CAN RX ISR 只负责入队，协议解析和控制在任务上下文执行
- 启动阶段使用 25 s IWDG，初始化成功后切换为 6 s 运行超时；监督任务仅在采样安全周期持续推进时喂狗

## 工程结构

```text
Core/          STM32CubeMX 生成的应用入口、外设初始化和 FreeRTOS 任务创建代码
User/          BMS 应用层、业务逻辑层、硬件适配层和设备驱动代码
Drivers/       STM32 HAL 与 CMSIS 库
Middlewares/   FreeRTOS 中间件
MDK-ARM/       Keil 工程文件
```

## 构建方式

使用 Keil uVision5 打开 `MDK-ARM/FREE-BMS.uvprojx`，然后执行全量编译。

当前 `main` 所含安全修复已使用 Keil uVision5 / ARM Compiler 5.06 update 7 完成全量重建，结果为 `0 Error, 0 Warning`。

## 烧录与调试

- 通过 ST-Link 使用 SWD 接口烧录，SWD 引脚为 PA13/PA14。
- 使用 USART1 作为调试串口，波特率为 115200。
- 串口输出用于查看 BQ 初始化、阈值回读、采样、保护状态、ALERT 事件和 UART 命令行交互。

## 开发与验证边界

- `Core/` 下的 CubeMX 生成文件只建议在 `USER CODE` 区域内修改。
- `User/` 集中放置 BMS 应用、业务、硬件适配和驱动代码；任何保护阈值、MOS、ALERT 或通信修改都需要重新编译并完成板端验证。
- 当前项目定位为固件功能与安全链路验证，不代表量产认证完成；安全修复仍需进行板端/HIL 故障注入测试，禁止通过真实电池短路进行验证。
